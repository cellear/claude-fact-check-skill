# Contributing to Social Media Fact-Check Skill

Thank you for considering contributing to this project! This fact-checking methodology is designed to help people verify claims more effectively, and community input makes it better.

## How to Contribute

### Reporting Issues

If you find problems with the methodology:

1. **Check existing issues** to see if it's already reported
2. **Open a new issue** with:
   - Clear description of the problem
   - Example of where the methodology falls short
   - Suggested improvement (if you have one)

### Suggesting Enhancements

We welcome suggestions for:

- Additional rhetorical fallacies to detect
- Better ways to assess source independence
- Improved output formatting
- New use cases or examples
- Clearer documentation

### Submitting Changes

1. **Fork** the repository
2. **Create a branch** for your changes (`git checkout -b improve-fallacy-detection`)
3. **Make your changes** to the relevant files
4. **Test** your changes by using the modified skill on real examples
5. **Commit** with clear messages (`git commit -m "Add detection for appeal to nature fallacy"`)
6. **Push** to your fork (`git push origin improve-fallacy-detection`)
7. **Open a Pull Request** with:
   - Description of what changed and why
   - Examples demonstrating the improvement
   - Any limitations or trade-offs

## Improvement Areas

### High Priority

- **Source Independence Analysis**: This is the most critical component. Improvements to citation chain detection are highly valuable.
- **Rhetorical Fallacy Detection**: Additional fallacies and better descriptions welcome.
- **Example Fact-Checks**: Real-world examples help users understand the methodology.

### Medium Priority

- **Documentation**: Clearer instructions, better formatting, more use cases.
- **Output Format**: Ways to make reports clearer or more actionable.
- **Triage Criteria**: Better ways to categorize claims.

### Guidelines for Changes

#### For SKILL.md Changes

- Keep the process sequential and clear
- Maintain emphasis on source independence (it's critical!)
- Be aggressive in fallacy detection (that's a feature, not a bug)
- Include examples where helpful
- Preserve the structured output format

#### For Documentation

- Use clear, accessible language
- Include concrete examples
- Acknowledge limitations honestly
- Keep instructions practical

#### For Examples

- Use real or realistic claims
- Show both successful and challenging fact-checks
- Demonstrate the full methodology
- Include diverse claim types (politics, health, science, etc.)

## Style Guidelines

- Use markdown formatting consistently
- Keep tone professional but accessible
- Be specific rather than vague
- Acknowledge uncertainty where appropriate
- Prioritize clarity over brevity

## Testing Your Changes

Before submitting:

1. **Try the modified methodology** on at least 2-3 different claims
2. **Check that all steps** still flow logically
3. **Verify the output** is clear and actionable
4. **Consider edge cases** (opinion claims, unfalsifiable claims, etc.)

## Code of Conduct

This project aims to help people think critically and avoid misinformation. Please:

- Be respectful and constructive in discussions
- Focus on methodology improvement, not political debates
- Acknowledge when you're uncertain
- Welcome feedback and different perspectives
- Remember that fact-checking is hard and we're all learning

## Questions?

Not sure if your idea is a good fit? Open an issue to discuss it first! We're happy to provide feedback before you invest time in changes.

## Recognition

Contributors who make significant improvements will be acknowledged in the README.
